map USB (vì đây là 1 mục lớn nên mình sẽ cho nó thành 1 phần)

+ Hệ điều hành windows ( amd and intel )

 Mở USB toolbox ( tải theo links Releases · USBToolBox/tool · GitHub ) và làm theo ảnh ( ảnh nà‌y mình lấy từ nguồn vietnamitx.com )   

Ở bước này các bạn cắm USB vào từng cổng

Sau đó các bạn chọn các port cần thiết bằng câu lệnh 1,2,3,….v.v ( 1,2,3 tương ứng với mã port) sau khi chọn xong các bạn chọn type bằng lệnh T:1,3,…:3 ( hoặc T:1:0 xem chi tiết type port kèm mã ở đây ) nhấn phím “K” để built kext ( tuyệt đối không chọn tất cả các port nếu các bạn chọn tất cả các port thì sẽ không sleep được )

Sau khi built xong các bạn bỏ kext vừa built vào mục kext trong efi kèm theo kext USBtoolbox.kext theo nguồn sau  Releases · USBToolBox/kext · GitHub tiếp theo các bạn snapshot lại config và restart lại máy và tận hưởng thành quả 

Lưu ý cách này cũng dùng được trên winpe và macos ( xem phần dưới )

Lưu ý 2: đối với các bạn dùng winpe để map thì ko nên dùng bộ anhdv boot trừ khi bạn có bản prenium khuyến khích dùng bộ nhv-boot hoặc hirren-boot ( links hirren-boot: Download | Hiren’s BootCD PE (hirensbootcd.org) | links nhv-boot: NHV BOOT 2021 BỘ CÔNG CỨU HỘ MẠNH MẼ , CHUYÊN NGHIỆP )

Lưu ý 3: đối với các bạn thực hiện theo cách này thì vẫn chưa sleep được đâu các bạn phải fix gprw/uprw/lanc

+ Hệ điều hành macos (do có nhiều cách nên mình sẽ liệt kê ra 2 cách mình thấy trực quan nhất nhé ) :

Cách 1: Sử dụng công cụ USB mapping của dortarnia (intel )

B1: Các bạn down các kext USBinjetall và xhciunsported theo nguồn sau 

Releases · Sniki/OS-X-USB-Inject-All · GitHub

xhci unsupported sẽ ở chung 1 folder với USB inject all

           GitHub – Sniki/OS-X-USB-Inject-All: Kext to inject all USB ports for the installed Intel EHCI/XHCI chipset automatically.

Hoặc theo nguồn Video Guide – USB Port Mapping Guide 2021 – Big Sur 11.3 with no Xhci Port Limit ( khuyến khích theo nguồn này )

           B2: các bạn tải corpnewt/USBMap: Python script for mapping USB ports in macOS and creating a custom injector kext.

 nhấn D sau đó cắm USB vào từng cổng 

B3: Ở bước này các bạn nhấn p để được như hình trên

B4: Gõ lệnh T:1,3,4,5,6,7,16,17,19,21,22:3 này để chọn các cổng USB 3.0 (lệnh trên mang tính chất minh họa )

B5: Gõ lệnh T:9:255 để tích hợp card bluetooth vào hệ thống ( lệnh trên chỉ mang tính chất minh họa ) 

B6: Nhấn k để built kext 

B7: Bỏ kext vào mục kext trong efi rồi snaps config và restart

Lưu ý : Không sử dụng chung SSDT-UIAC.aml hoặc USBInjectAll với USBmap.kext có thể sử dụng  Xhci Unsupported  nếu cần

Lưu ý 2: nếu các bạn map bằng usb mapping mà gặp lỗi như sau

Thì các bạn có thể sử dụng ssdt-usb-rest để fix hoặc các bạn có thể dùng hot patch usb các mục mà usb mapping yêu cầu cách này mình có hướng dẫn ở mục XV

Cách 2 : Sử dụng hackintool để map USB ( intel and amd )

  • Chuẩn bị : 
  • 1 USB 3.0 1‌USB 2.0 ( có thể sử dụng cap của chuột) 1u‌sb type c ( nếu cần có thể dùng  cap của ổ cứng rờ‌i ) 
  • kext USB inject all và xhci ‌‌unsported ( lấy ở trên ) vào config tắt xhci limi‌ted đi trong ker‌‌nel Và đương nhiên không thể thiếu hackintool có thể lấy t‌heo đường dẫn sau hoặc ở mục các tool hackint‌osh trong folder opencore của mình Releases · headkaze/Hackintool · GitHub hoặc CÁC TOOL HACKINTOSH  
  • Tiến hàn‌h ( ở đây mình chia làm 2 phần là remap USB 2.0 và remap USB 3.0 )
  • Phần 1 ( remap USB 2.0 ) :

B1: các bạn cắm 1 chiếc USB 2.0 vào tất cả cổng trong hệ thống ( mở hakcintool tap usb )

B2: ở đây các cổng thực sẽ hiện màu xanh các bạn hãy ghi nhớ phần name của các cổng ko hiện xanh ( có thể gh‌i tên các cổng đây ra )

B3: các bạ‌n thêm đoạn code sau vào boot-arg 

uia_exclude= “tên các cổng không hiện xanh” ( khi viết code không có dấu “ ”) 

Đây là đoạn code của mình sau khi viết ra các bạn nhớ thêm vào boot-arg nhé

            B4: Restart và xem tiếp phần 2

  • Phần 2 ( remap USB 3.0, type C ) :

B1: Các bạn cắm 1 chiếc USB 3.0 và USB typ‌e c vào tất cả cá‌c cổng trên hệ thống 

B2: Các bạn sửa lại đoạn code trên theo đúng với hiện tại ( loại bỏ 1 số cổng vừa hiện xanh trong code ) tiếp các bạn xóa đoạn code ở phần 1 đi và dán đoạn code mới vào boot-arg 

B3: Restart máy và tiếp tục xem 

Sau khi các bạn đã làm xong cả 2 phần trên thì hãy điều chỉnh lại loại cổng trong hackintool cho đúng nhé 

  • cổng t‌‌ype c các bạ‌n điều chỉnh là typec 
  • cổng USB 3.0 điều chỉnh lại là 3.0 
  • Usb bluetooth và camera các bạn điều chỉnh lại là ‌‌internal 

đây là của mình sau khi điều chỉnh xong 

Lưu ý‌: Hãy chắc chắn rằng bạn đã loại bỏ tất cả port‌ rỗng ( đối với 1 số máy sẽ còn lại 1 hoặc 2 port rỗng nhưng không sau bạn có thể tiếp tục )

Sau khi làm xong các bước ở trên bạn nhấn vào nút để xuất ra 2 file  ssdt và 1 kext tiếp theo bạn xóa các file đấy đi chỉ giữ lại file ssdt-uiac thôi ( tức là xóa file ssdt-e‌c-USBx và file USB port.kext)

Sau khi làm xong các bạn copy file ssdt uiac vào mục acpi và snaps lại tiếp các bạn restart bây giờ các bạn đã có thể xóa các code trong boot-arg được rồi

Lưu ý 2 : các bạn có thể để load cùng với  USBInjectAll và  Xhci Unsupported  nếu cần

Lưu ý 3 : nếu khi remap usb 2.0 xong mà khi cắm vào các cổng usb 3.0 không nhận thì bạn hãy gộp remap usb 2.0 và remap usb 3.0 lại thành remap usb 

Custom usbinjects all

đối với 1 vài mấy cần hotpatch usb ( xem ở mục dưới ) và thêm id vào usb injects all ( chỉ đối với gen 10 hoặc gen 11 ) làm như sau

B1: mở hackintool lên vào tab usb 

B2: nhìn lên mục đầu tiên ta sẽ thấy id usb ( id usb = vendor + device id vd id của mình là 80861e31 | các port EH0X không cần add id )

B3: tải kext usb inject all sau đó chọn show package content

B4: mở file info.plist lên và add id vừa lấy được ở trên đổi tên 1 id gen 9 thành id của bạn 

  • đổi tên 1 số port bị nhầm tên  ( làm cũng được không làm cũng không sao )

cách này dành cho 1 số bạn có tính cầu toàn và yêu cần sự hoàn hảo thật ra nó chả có công dụng gì nhiều ngoài việc cho đẹp

B1: chuẩn bị 1 usb3.0 và 1 usb 2.0

B2: tải hackintool

B3: cắm usb 2.0 vào port 2.0 cắm usb 3.0 và cùng 1 port với usb 2.0

B4: nếu nó khác tên thì các bạn đổi tên lại trong info.plist nếu giống tên thì không cần đổi 

Lưu ý: Xem chi tiết cách này tại đây ( mình có hướng dẫn cách đọc dsdt )

Cách 3 sử dụng us btoolbox ( intel and amd )

B1: các bạn tải kext sau về Releases · USBToolBox/kext (github.com) ( bỏ cả 2 kext vào EFI ⇒ OC )

B2: restart máy sau đó tải tool sau về USBToolBox/tool: the USBToolBox tool (github.com)

B3: mở usb toolbox lên các bạn sẽ được như sau 

B4: nhấn D cắm usb vào hết tất cả các port

B5: nhấn b sau đó nhấn s

B6: chỉnh lại type cho các usb 

  • ấn T để xem loại type hoặc truy cập vào links sau tool/TYPES.md at master · USBToolBox/tool (github.com)
  • nhập lệnh sau T:số port: loại type ( để dễ xem port thì các bạn nên làm như sau 
  • bước 1 ngay màn hình chọn các loại option các bạn chọn C
  • Bước 2 các bạn chọn T sau đó ấn B )

vd: T:1,2:0 ( port 1,2 là loại usb 2 )

T:3:3 ( port 3 là loại usb 3 )

T:4,5:255 ( port 4,5 là internal )

B7: nhấn K để built kext 

B8: cho kext vào EFI ⇒ OC ⇒ kext ( hãy chắc chắn rằng bạn đã xóa UTBDefault.kext đi và đảm bảo trong kext có 2 kext usbtoolbox.kext và UTBMap.kext 

B9: snaps lại

B10: restart và tận hưởng thôi

 

source tham khảo :USBToolBox/tool: the USBToolBox tool (github.com)

+ Hotpatch usb

B1: Cho kext usbinjectall của rehabman theo links sau RehabMan / OS-X-USB-Inject-All / Downloads — Bitbucket ( có thể dùng kext của snikin)

B2: Các bạn mở config bằng propertree và add các dòng sau vào phần acpi ⇒ patch 

0.(Số này sẽ được thay đổi tùy theo config của mỗi người )

1.

2.

Replace | data | 58484349

B3: Save lại tiếp snaps và reboot

B4: Các bạn tải app sau về để check IO Registry Explorer

Lưu ý : nếu dùng opencore các bạn cần add đầy đủ các mục như hình

Lưu ý 2: Sau khi làm như này xong các bạn vẫn chưa nhận được web cam (tùy người) nếu như vậy các bạn có thể map lại usb bằng usb mapping hoặc hackintool (có hướng dẫn ở trên)
